# Dialogue Quality Analyzer

## Выбор инструмента
Использована модель GPT‑4o‑mini через OpenAI API для анализа диалогов. 
Она недорогая, достаточно быстро работает за счет API (локально у меня нет GPU, а на CPU LLM работает медленнее), 
хорошо распознает ключевые темы и возражения и возвращает достаточно качественно структурированный JSON.

## Ограничения
1. Зависимость от стабильности сети. Кроме того, подключение к API вынуждает шифровать данные перед отправкой запроса в LLM.
2. Иногда при "manager_handled": false модель возвращает пустой список "manager_actions". Может быть немного критично в некоторых задачах.

## Следующие шаги
1. При наличии GPU можно перейти на локальную LLM для ещё более быстрой работы. Аналогами GPT‑4o‑mini могут быть Mistral-Small-3 и Llama-3-8B-Instruct.
2. Добавить нормализацию регистра и пустых полей для единообразного JSON.
3. Создать API-обёртку или веб-интерфейс для удобного запуска анализа. Или интегрировать с внутренними системами CRM для анализа в реальном времени и визуализации результатов.

## Инструкция по запуску
- Добавить файл .env и сохранить в него OPENAI_API_KEY
- Создать и активировать виртуальную среду
- Установить необходимые зависимости из `requirements.txt`
```
pip install -r requirements.txt
```
- Запустить скрипт `analyzer.py`
```
python analyzer.py
```

Результат будет в консоли и сохранится в файл `analysis_results.json`
